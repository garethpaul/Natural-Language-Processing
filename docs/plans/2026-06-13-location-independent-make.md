# Location-Independent NLP Verification

status: completed

## Context

The Make recipes resolve source, test, checker, and cleanup paths from the
caller's working directory. An absolute Makefile invocation from elsewhere can
fail, inspect unrelated files, or delete generated bytecode outside the
checkout; its recursive final clean also loses the selected Makefile.

## Objectives

- Resolve cleanup, compilation, tests, static checks, and recursive cleanup
  from the checkout containing the loaded Makefile.
- Preserve the existing target graph and `PYTHON` override.
- Enforce the exact rooted recipes, operator guidance, completed status, and
  verification evidence in the active baseline checker.
- Prove root and external-directory behavior plus cleanup containment with
  mutation-sensitive checks.

## Implementation Units

### Make Contract

Files: `Makefile` and `scripts/check-baseline.py`.

Derive one absolute checkout root, run path-sensitive commands from that root,
scope cleanup to it, and pass the absolute Makefile to the recursive clean.
Require the complete small Makefile so those guarantees cannot drift.

### Documentation And Evidence

Files: `README.md`, `CHANGES.md`, and this plan.

Document absolute Makefile invocation and record root, external, cleanup, and
hostile-mutation verification after it completes.

## Boundaries

- Do not change detector logic, tests, dependency files, workflows, or data.
- Do not install packages, download corpora, use private text, or access the
  network.
- Preserve the existing stacked PR chain and exact-head evidence.
